# Security Policy

## Supported Versions

The following versions of Studytrix are currently being supported with security updates:

| Version | Supported |
| :--- | :--- |
| 1.0.x | Yes |
| < 1.0.0 | No |

## Reporting a Vulnerability

The Attendrix Team takes the security of our users' data and credentials seriously. If you believe you have found a security vulnerability in Studytrix, please report it through the following process:

1. **Do Not Open a Public Issue**: Please do not share security-sensitive information in a public GitHub issue.
2. **Private Reporting**: Send a detailed report to the maintainers at [security@attendrix.org](mailto:security@attendrix.org) or contact a senior member of the Attendrix Team directly.
3. **Information to Include**: Include as much information as possible, including:
    - Description of the vulnerability.
    - Steps to reproduce.
    - Potential impact.
    - Possible mitigations.

## Out of Scope

Please note that Studytrix is a proxy layer. Issues related to the upstream Google Drive API or local browser environments (e.g., local storage accessibility in compromised browsers) are generally considered out of scope, unless Studytrix-specific logic exacerbates the risk.

## Response Process

We aim to acknowledge all security reports within 48 hours and provide an initial assessment within 5 business days. We will keep you informed of our progress as we work to resolve the issue.

We thank you for your help in keeping Studytrix secure.
